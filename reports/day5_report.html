2. Why Walk-Forward, Not a Random 80/20 Split?

⚠ The 80/20 Split Is Wrong for Time Series A random 80/20 split on financial data shuffles observations across time. Your "test" set contains data from 2015, but your training set contains 2020 data. The model trained on a COVID crash is being evaluated on a pre-crash market it has already seen. This is future leakage in reverse — and it inflates performance metrics just as badly as forward lookahead bias.

Walk-forward validation enforces strict temporal causality:

  • Every training set ends before its corresponding test set begins — no exceptions.
  • The model only ever sees the past to predict the future — exactly how live deployment works.
  • The test periods are consecutive and cover the full out-of-sample horizon — you get one continuous out-of-sample record, not a cherry-picked period.
✨ Industry Standard Walk-forward validation (also called "time-series cross-validation" or "anchored cross-validation") is the standard method for validating quantitative trading strategies. Any strategy validated with a random split is considered scientifically invalid in professional quant finance.

3. Walk-Forward Mechanics — Step by Step

3.1 The Fold Structure

initial_train_years=3, step_months=3

Fold 0: Train [2010-01 → 2012-12] │ Test [2013-01 → 2013-03]
Fold 1: Train [2010-01 → 2013-03] │ Test [2013-04 → 2013-06]
Fold 2: Train [2010-01 → 2013-06] │ Test [2013-07 → 2013-09]
...
Fold 43: Train [2010-01 → 2024-09] │ Test [2024-10 → 2024-12]

Total: (14 years − 3 year init) × 4 folds/year = 44 folds
All test periods concatenated = one continuous out-of-sample record from 2013–2024

3.2 Expanding Window vs. Rolling Window

Expanding Window (our choice)

Training set grows with each fold. Fold 43 trains on 14 years of data. More data = better model in general.

Use when: you believe all historical data is relevant — no concept drift.

Rolling Window (alternative)

Training set is a fixed-length window that slides forward. Fold 43 might train on only the last 3 years.

Use when: you believe old data hurts (regime has structurally changed).

We use expanding because financial return distributions, while noisy, do not fundamentally change over the 2010–2024 period. More data is strictly better for our XGBoost and LSTM agents.

3.3 Code Walkthrough

def walk_forward_backtest(agents, aggregator, feature_df,
                          initial_train_years=3, step_months=3):
    aggregator.reset()   # always start with uniform weights
    agent_names = [a.name for a in agents]

    start = feature_df.index[0]
    # train_end = 2012-12-31 for initial_train_years=3
    train_end = start + relativedelta(years=initial_train_years) - relativedelta(days=1)

    records = []
    fold = 0

    while True:
        test_start = train_end + relativedelta(days=1)
        test_end   = train_end + relativedelta(months=step_months)
        if test_start > feature_df.index[-1]:
            break   # no more data to test on
        test_end = min(test_end, feature_df.index[-1])

        train_df = feature_df.loc[:train_end]
        test_df  = feature_df.loc[test_start:test_end]

        # Step 1: fit all agents on the expanding training set
        for agent in agents:
            agent.fit(train_df)

        # Step 2: pre-compute all agent predictions for the test window
        # (more efficient than calling predict() per day)
        agent_preds_all = {name: agent.predict(test_df)
                           for name, agent in zip(agent_names, agents)}

        # Step 3: feed predictions day-by-day to Hedge (true online learning)
        for i, (date, row) in enumerate(test_df.iterrows()):
            actual = row["log_return"]
            step_preds = [float(agent_preds_all[name][i]) for name in agent_names]

            ensemble = aggregator.aggregate(step_preds)  # weighted average
            aggregator.update(step_preds, actual)         # update weights NOW
            # ^ Hedge sees the truth and adjusts before predicting tomorrow

            record = {"date": date, "actual": actual,
                      "ensemble_pred": ensemble, "fold": fold}
            for name, pred in zip(agent_names, step_preds):
                record[f"{name}_pred"] = pred
            for name, w in zip(agent_names, aggregator.weights):
                record[f"{name}_weight"] = w
            records.append(record)

        train_end += relativedelta(months=step_months)  # expand train window
        fold += 1

    return pd.DataFrame(records).set_index("date")

4. How Hedge Updates Within Each Fold

💡 Key Insight: Per-Day Updates, Not Per-Fold The Hedge aggregator updates its weights after every single trading day — not once per fold. This is what makes it true online learning. After observing Monday's return, weights shift before Tuesday's prediction. The agents refit only quarterly (per fold), but the aggregator adapts daily.

Why This Matters

Consider the COVID crash of March 2020. A fold might cover Jan–Mar 2020. If Hedge only updated once per fold, it would use the weights from January (calm market) throughout the crash, missing the signal that VolatilityAgent suddenly became much more accurate than TrendAgent.

With per-day updates:

  • Day 1 of the crash: VolatilityAgent predicts correctly, TrendAgent fails → VolatilityAgent weight increases
  • Day 2: VolatilityAgent now has more weight, ensemble is better positioned
  • Week 2: weights have substantially shifted toward regime-aware agents

This is the regime-adaptive behavior that makes the Hedge ensemble more than just a simple average.

The Pre-Computation Pattern

# All predictions cached upfront for efficiency
agent_preds_all = {name: agent.predict(test_df)
                   for name, agent in zip(agent_names, agents)}

# Then fed day-by-day to the Hedge loop
step_preds = [float(agent_preds_all[name][i]) for name in agent_names]

Calling agent.predict(test_df) once per fold is far more efficient than calling it 63 times (one per trading day in a quarter). XGBoost and LSTM batch prediction is vectorized — no overhead per-row.

5. Metric 1 — Sharpe Ratio

What it measures: Risk-adjusted return. How much return are you getting per unit of risk (volatility) taken? The higher, the better.

SR = mean(r) / std(r) × √252

Why √252?

The formula computes daily statistics (mean and std of daily returns), then annualizes them. There are 252 trading days per year. The mean scales linearly with time (252× for annual), while standard deviation scales with √time (√252 for annual). Dividing mean by std and then multiplying by √252 gives the annualized Sharpe ratio.

Interpretation Scale

Sharpe RatioQualityContext
Below 0Losing money risk-adjustedWorse than holding cash
0 – 0.5WeakTypical passive equity exposure
0.5 – 1.0AcceptableSPY buy-and-hold historically ≈ 0.5–0.8
1.0 – 2.0GoodStrong discretionary fund
2.0 – 3.0ExcellentTop quantitative strategies
Above 3.0Exceptional (or overfitted)Renaissance Medallion Fund (~3+)
 
def sharpe_ratio(returns, annualization=252):
    returns = np.asarray(returns, dtype=float)
    if returns.std() == 0:
        return 0.0   # guard: all predictions identical → no meaningful Sharpe
    return float(returns.mean() / returns.std() * np.sqrt(annualization))

6. Metric 2 — Max Drawdown

What it measures: The largest peak-to-trough decline in the equity curve. Represents the worst loss an investor following this strategy would have experienced at any point. Expressed as a positive fraction (0 = no drawdown, 1 = total loss).

MaxDD = max over t of (peak_t − value_t) / peak_t

where peak_t = max(value_0, value_1, ..., value_t) ← running maximum

Why np.maximum.accumulate?

def max_drawdown(equity_curve):
    curve = np.asarray(equity_curve, dtype=float)

    # np.maximum.accumulate: at each index t, gives max(curve[0..t])
    # This is the "all-time high" at each point in time
    peak = np.maximum.accumulate(curve)

    # drawdown[t] = how far below the all-time high are we right now?
    drawdown = (peak - curve) / peak

    return float(drawdown.max())   # worst point across all time

The Equity Curve Construction

# strategy_returns = sign(pred) * actual — one value per day
equity_curve = 10_000 * np.exp(np.cumsum(strategy_returns))
#              └─ starting capital                              
#                           └─ cumulative sum of log returns = total log return to date
#                    └─ exp() converts log return back to price multiplier

Starting with $10,000: if daily log returns compound to a total of +0.5 (50% gain), the equity curve peaks at $10,000 × e^0.5 ≈ $16,487.

📋 Why Sharpe Alone Is Insufficient A strategy can have SR = 1.5 (looks great) but 60% max drawdown (catastrophic in practice). No real investor can tolerate losing 60% of capital, even temporarily. Max drawdown catches strategies that work on average but blow up in stress regimes — exactly the regimes the Volatility and Sequence agents are designed to handle.

7. Metric 3 — Information Ratio

What it measures: How consistently you outperform a benchmark (buy-and-hold SPY). While Sharpe measures absolute risk-adjusted return, IR measures skill relative to a reference.

IR = mean(r_strategy − r_benchmark) / std(r_strategy − r_benchmark) × √252

= Active Return / Tracking Error × √252
TermFormulaWhat it means
Active Returnmean(strategy − benchmark)Average daily outperformance vs. buy-and-hold
Tracking Errorstd(strategy − benchmark)Consistency of that outperformance — how much it varies
IRActive / Tracking × √252Risk-adjusted outperformance

Interpretation

  • IR > 0.5: Good — considered acceptable in active fund management
  • IR > 1.0: Excellent — rare in practice; implies consistent alpha
  • IR < 0: You are underperforming buy-and-hold on a risk-adjusted basis

Sharpe vs. IR — The Key Difference

A strategy that loses money every day but less than the market (during a crash) has a positive IR but negative Sharpe. A pure long strategy that never shorts might have a positive Sharpe but near-zero IR vs. SPY. They measure different aspects of performance, and both belong in a complete evaluation.

8. The Strategy Return Construction

The key line: strategy_returns = np.sign(y_pred) * y_true

This models a simple long/short trading signal: go long (buy) when you predict up, go short (sell) when you predict down. Position size is always 1 (no scaling by conviction).

PredictionActualPositionStrategy ReturnResult
pred > 0 (↑)actual > 0 (↑)Long+actualCorrect — earn full gain
pred > 0 (↑)actual < 0 (↓)Long−|actual|Wrong — lose full loss
pred < 0 (↓)actual < 0 (↓)Short+|actual|Correct — profit from decline
pred < 0 (↓)actual > 0 (↑)Short−actualWrong — lose on short squeeze

This is a unit-position long/short strategy. The Sharpe ratio of this strategy measures how well directional accuracy translates to risk-adjusted profit. An agent with 53% directional accuracy will have positive strategy returns on average, since it earns on 53% of days and loses on 47%.

💡 What np.sign(0) Returns np.sign(0.0) = 0.0 — a zero prediction means no position is taken, earning 0 return. This is correct: if the model has no view, it should be flat (no position). In practice, log returns of exactly 0 are extremely rare on daily data.

9. full_metrics_report — What It Compares

def full_metrics_report(results_df, baseline, agent_names):
    bh_returns = baseline.loc[results_df.index].values

    tables = []
    # 1. Each individual agent
    for name in agent_names:
        tables.append(metrics_table(results_df["actual"].values,
                                    results_df[f"{name}_pred"].values,
                                    benchmark=bh_returns, label=name))

    # 2. Hedge ensemble (adaptive weights)
    tables.append(metrics_table(results_df["actual"].values,
                                results_df["ensemble_pred"].values,
                                benchmark=bh_returns, label="Hedge Ensemble"))

    # 3. Equal-weight ensemble (fixed 1/N weights — Hedge baseline)
    eq_pred = results_df[[f"{n}_pred" for n in agent_names]].mean(axis=1).values
    tables.append(metrics_table(results_df["actual"].values, eq_pred,
                                benchmark=bh_returns, label="Equal Weight"))

    # 4. Buy-and-hold (always long signal)
    tables.append(metrics_table(results_df["actual"].values,
                                np.abs(results_df["actual"].values),
                                benchmark=bh_returns, label="Buy & Hold"))

    combined = pd.concat(tables, ignore_index=True)
    return combined.sort_values("Sharpe Ratio", ascending=False).reset_index(drop=True)

11. Full Interview Q&A

Q1. What is walk-forward backtesting and why is it the correct method for financial time series?
Walk-forward backtesting trains a model on all data up to time t and tests it on data after t, then advances t forward and repeats. This strictly enforces that no future information is used during training — exactly mirroring live deployment. A random 80/20 split on time series data is invalid because it allows the model trained on 2020 data to be tested on 2015 data, which is temporally impossible in production. Walk-forward is the industry standard for quantitative strategy validation.
Q2. What is the Sharpe ratio and what does it measure?
The Sharpe ratio measures risk-adjusted return: how much return you earn per unit of volatility taken. Formula: SR = mean(r) / std(r) × √252 for daily returns. A strategy earning 8% annually with 4% volatility has SR=2; one earning 8% with 16% volatility has SR=0.5. The first is dramatically better — same return, much less risk. SR > 1 is considered good; SR > 2 is excellent; above 3 is exceptional (or a sign of overfitting).
Q3. Why do you multiply by √252 in the Sharpe ratio formula?
We compute daily statistics (mean and std of daily returns) but report an annualized figure so it's comparable across strategies with different rebalancing frequencies. Mean scales linearly with time — annual mean = 252 × daily mean. Standard deviation scales with the square root of time — annual std = √252 × daily std. When you divide mean by std and multiply by √252, you get the annualized Sharpe ratio. This is the standard convention.
Q4. What is max drawdown and why is it important alongside Sharpe ratio?
Max drawdown is the maximum peak-to-trough decline in the equity curve: the worst loss an investor would have experienced at any point. A strategy can have a high Sharpe ratio on average but exhibit catastrophic drawdowns in stress periods. SPY lost 57% in the 2008 financial crisis — no institutional investor can tolerate that without redemptions. We use max drawdown alongside Sharpe to catch strategies that work on average but blow up in tail events, which is precisely the regime the VolatilityAgent is designed to handle.
Q5. What is the Information Ratio and how does it differ from the Sharpe ratio?
The Information Ratio measures skill relative to a benchmark: IR = mean(strategy − benchmark) / std(strategy − benchmark) × √252. Active return is how much you outperform buy-and-hold on average; tracking error is how consistently you outperform. The Sharpe ratio measures absolute risk-adjusted return — it doesn't compare you to anything. A strategy might have high Sharpe (makes money with low volatility) but negative IR (still underperforms buy-and-hold). Both metrics are needed for a complete picture.
Q6. Why do you construct strategy returns as sign(pred) × actual?
This models a unit long/short signal: go long (position = +1) when predicting a positive return, go short (position = −1) when predicting negative. The actual P&L for that day is position × actual_return. When correct, you earn the full magnitude of the move. When wrong, you lose it. Position size is always 1 — we are not scaling by prediction magnitude, just using direction. This is the simplest tradeable interpretation of our regression predictions.
Q7. What is the difference between expanding window and rolling window walk-forward?
In an expanding window, the training set grows with each fold — fold 43 trains on all 14 years of data. In a rolling window, training is a fixed-length window that slides forward — fold 43 might train on only the last 3 years. We use expanding because we believe all historical financial data is informative and more data improves model quality. Rolling window is preferred when you suspect concept drift — when old data from a different regime actively hurts predictions.
Q8. Why do Hedge weights update per day rather than per fold?
Because that is what online learning means: observe the outcome, update immediately, predict again. If weights only updated per fold (quarterly), the aggregator could not adapt during market regime changes within a fold — like the COVID crash in March 2020 (within the Jan–Mar 2020 test window). Per-day updates let the ensemble detect in real time which agents are handling the current regime well and shift weight to them, which is the theoretical guarantee Hedge provides.
Q9. What does the buy-and-hold baseline tell you?
Buy-and-hold represents the passive alternative: invest in SPY and do nothing. If our ensemble can't beat a passive investor, there is no value in the complexity of 4 agents, an LSTM, and a Hedge aggregator. The buy-and-hold Sharpe ratio on SPY historically runs 0.5–0.8 — that is our minimum bar. Any ensemble that scores below this has failed. The Information Ratio specifically measures how much better (or worse) we are than this baseline on a risk-adjusted basis.
Q10. What is tracking error?
Tracking error is the standard deviation of the difference between strategy returns and benchmark returns: std(strategy − benchmark). It measures how far your strategy's day-to-day performance deviates from the benchmark. Low tracking error means your returns closely follow the benchmark (even if on average you do better). High tracking error means you go your own way — sometimes better, sometimes worse. The Information Ratio divides active return by tracking error to give a signal-to-noise ratio of your outperformance.
Q11. If your ensemble has a higher Sharpe than buy-and-hold but a lower Information Ratio, what does that mean?
It means the ensemble generates a good absolute risk-adjusted return, but on average it is not consistently outperforming buy-and-hold. This can happen when the strategy has low volatility (good Sharpe) but also low active return relative to tracking error. In practice, this would mean our strategy is valuable for conservative investors (low risk), but would underperform a passive SPY investor after accounting for transaction costs. It would motivate increasing the signal quality of directional predictions rather than just reducing volatility.
Q12. How many backtest folds does your engine produce?
With 14 years of data (2010–2024), a 3-year initial training window, and 3-month steps: the first test period starts January 2013, and the last completes December 2024. That is 11 years × 4 quarters = 44 folds. Each fold contributes approximately 63 trading days (one quarter), giving a total out-of-sample record of roughly 44 × 63 ≈ 2,750 daily observations — a statistically meaningful evaluation set.
