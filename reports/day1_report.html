2. File 1 — data/fetch.py

2.1 What It Does

  • Downloads SPY (S&P 500 ETF) daily OHLCV data from 2010-01-01 to 2024-12-31 via yfinance
  • Downloads VIX (CBOE Volatility Index) for the same period
  • Computes log returns: log_return = ln(Close_t / Close_{t-1})
  • Joins both datasets on the date index, drops missing rows
  • Saves to data/spy_features.csv and data/vix.csv
  • Exposes load_or_fetch() — reads from CSV cache if available, downloads fresh if not

2.2 Full Code + Explanation

import pathlib
import numpy as np
import pandas as pd
import yfinance as yf

# pathlib.Path(__file__).parent → absolute path of the directory this script lives in
# The / operator on Path objects does path joining (not division)
DATA_DIR = pathlib.Path(__file__).parent
SPY_CSV  = DATA_DIR / "spy_features.csv"
VIX_CSV  = DATA_DIR / "vix.csv"

START = "2010-01-01"
END   = "2024-12-31"


def fetch_spy() -> pd.DataFrame:
    df = yf.download("SPY", start=START, end=END,
                     auto_adjust=True, progress=False)
    # auto_adjust=True: adjusts for stock splits & dividends
    # Without this, a 2-for-1 split looks like a 50% price drop overnight

    df = df[["Close"]].copy()   # .copy() prevents SettingWithCopyWarning
    df.columns = ["Close"]

    # KEY LINE: log return = ln(today's close / yesterday's close)
    # .shift(1) gives the previous row's Close — so this is NOT lookahead
    df["log_return"] = np.log(df["Close"] / df["Close"].shift(1))

    df.dropna(inplace=True)   # first row has NaN (no previous close)
    return df


def fetch_vix() -> pd.DataFrame:
    vix = yf.download("^VIX", start=START, end=END,
                      auto_adjust=True, progress=False)
    vix = vix[["Close"]].copy()
    vix.columns = ["vix_close"]

    # 5-day percentage change in VIX: is fear rising or falling?
    vix["vix_change_5d"] = vix["vix_close"].pct_change(5)

    vix.dropna(inplace=True)
    return vix


def build_dataset() -> pd.DataFrame:
    spy = fetch_spy()
    vix = fetch_vix()

    # .join() aligns on shared DatetimeIndex
    # how="inner" keeps only dates present in BOTH datasets
    df = spy.join(vix, how="inner")
    df.dropna(inplace=True)

    spy.to_csv(SPY_CSV)      # save to disk for caching
    vix.to_csv(VIX_CSV)
    return df


def load_or_fetch() -> pd.DataFrame:
    # Cache pattern: read CSV if exists (milliseconds), else download (5 seconds)
    if SPY_CSV.exists() and VIX_CSV.exists():
        spy = pd.read_csv(SPY_CSV, index_col=0, parse_dates=True)
        vix = pd.read_csv(VIX_CSV, index_col=0, parse_dates=True)
        return spy.join(vix, how="inner").dropna()
    return build_dataset()

2.3 Why Log Returns? (Not Prices)

📊 Non-Stationarity Problem

SPY was $115 in 2010, $480 in 2024. A model trained on 2010 prices has never seen $400+ inputs — pure distributional shift. Log returns are roughly stable: ~1% daily std, centered near 0, year after year.

➕ Additivity Property

Simple returns compound wrongly: -10% then +10% = -1%, not 0. Log returns add perfectly: r_total = r_1 + r_2 + ... + r_n. This makes multi-period aggregation trivial.

🇮🇲 Near-Gaussian Distribution

ML models (linear regression, SVR, LSTM) work best with approximately normal inputs. Log returns are far closer to Gaussian than raw prices, reducing bias in model learning.

🆕 Finance Theory Connection

Log returns arise from geometric Brownian motion — the foundation of Black-Scholes. Using them connects your ML pipeline to the standard quant finance mathematical framework.
log_return_t = ln( Close_t / Close_{t-1} ) ≈ (Close_t − Close_{t-1}) / Close_{t-1} for small moves

2.4 What Is VIX and Why Include It?

The VIX (CBOE Volatility Index) is the "fear gauge" of Wall Street. It is computed from the implied volatilities of S&P 500 options expiring in the next 30 days. It represents what the options market expects the S&P 500 to move over the next month (annualized).

  • VIX = 12: Calm market, ~0.75% expected daily move. Market is complacent.
  • VIX = 30: Stress regime. ~1.9% expected daily move.
  • VIX = 80: (March 2020, COVID) ~5% expected daily move. Extreme fear.

Why VIX and not just price volatility? VIX captures the options market's forward-looking fear. High VIX while realized price volatility is still low = early warning signal the options market has detected a threat before the price series has moved. They carry different information.

3.2 Algorithm 1: RSI (Relative Strength Index)

What it measures: Whether the asset is overbought (RSI > 70) or oversold (RSI < 30) relative to its recent history. Scale: 0–100.

Mathematical Formula

Step 1: delta_t = Price_t − Price_{t-1}

Step 2: gain_t = max(delta_t, 0)     loss_t = max(−delta_t, 0)

Step 3: avg_gain = EWM(gain, com=13)    avg_loss = EWM(loss, com=13)

Step 4: RS = avg_gain / avg_loss

Step 5: RSI = 100 − (100 / (1 + RS))

Code Walkthrough

def compute_rsi(series: pd.Series, period: int = 14) -> pd.Series:
    delta = series.diff()           # price[t] - price[t-1]

    gain = delta.clip(lower=0)     # keeps positives, zeros out negatives
    loss = -delta.clip(upper=0)    # keeps magnitudes of losses as positive numbers

    # EWM with com=13: alpha = 1/(1+13) = 1/14
    # min_periods=14: return NaN until we have 14 observations
    avg_gain = gain.ewm(com=period - 1, min_periods=period).mean()
    avg_loss = loss.ewm(com=period - 1, min_periods=period).mean()

    rs = avg_gain / avg_loss.replace(0, np.nan)  # avoid division by zero
    return 100 - (100 / (1 + rs))
💡 Why EWM, not simple rolling average? Two reasons: (1) EWM responds faster to regime changes — recent observations get higher weight. (2) It avoids the "drop-off cliff" — with SMA, when an extreme observation leaves the 14-day window, RSI jumps discontinuously even if nothing happened today. EWM never fully forgets; it just down-weights old observations.

ParameterFormulaExample

3.3 Algorithm 2: MACD (Moving Average Convergence/Divergence)

What it measures: Whether short-term momentum is accelerating (MACD positive) or decelerating (MACD negative) relative to the longer-term trend. Positive MACD = fast EMA above slow EMA = bullish short-term momentum.

EMA_fast = EWM(price, span=12) ← "where is the market right now?"

EMA_slow = EWM(price, span=26) ← "where has the market been?"

MACD_line = EMA_fast − EMA_slow

Signal_line = EWM(MACD_line, span=9) ← smoothed MACD (what we use as feature) 
def compute_macd(series: pd.Series, fast=12, slow=26, signal=9):
    # adjust=False uses recursive formula: EMA_t = α*price_t + (1-α)*EMA_{t-1}
    # This is the finance industry standard convention
    ema_fast    = series.ewm(span=fast,   adjust=False).mean()
    ema_slow    = series.ewm(span=slow,   adjust=False).mean()
    macd_line   = ema_fast - ema_slow
    signal_line = macd_line.ewm(span=signal, adjust=False).mean()
    return macd_line, signal_line
📋 Why use the Signal Line, not the MACD Line? The raw MACD line is noisy. The signal line (9-period EWM of MACD) is a smoother version. For ML models, noisy features add variance without adding predictive information, so we use the smoothed signal line as the feature.

3.4 Algorithm 3: Bollinger Band Width

What it measures: Market volatility as a normalized envelope around price. Narrow bands = quiet market (often precedes a large breakout). Wide bands = turbulent market. Uses SMA (not EWM) by original design.

Middle Band = SMA(price, 20)

Upper Band = Middle + 2 × std(price, 20)

Lower Band = Middle − 2 × std(price, 20)

BB Width = (Upper − Lower) / Middle = 4 × std / SMA 
def compute_bollinger(series: pd.Series, window: int = 20) -> pd.Series:
    mid   = series.rolling(window).mean()    # SMA (not EWM) — Bollinger's original spec
    std   = series.rolling(window).std()
    upper = mid + 2 * std
    lower = mid - 2 * std
    # Divide by mid: normalizes for price level (2010 SPY=$115 vs 2024 SPY=$480)
    return (upper - lower) / mid             # simplifies to 4*std/mid

3.5 Algorithm 4: Lag Features

def compute_lag_features(series: pd.Series, lags=(1, 2, 3, 5, 10)) -> pd.DataFrame:
    # Dictionary comprehension: creates one column per lag
    # lag_1 at row t = series[t-1] (yesterday's return)
    # lag_10 at row t = series[t-10] (two weeks ago)
    return pd.DataFrame(
        {f"lag_{lag}": series.shift(lag) for lag in lags}
    )

Why these specific lags? 1, 2, 3 capture short-term serial dependence. 5 = one week (weekly patterns). 10 = two weeks (medium-term momentum).

3.6 Algorithm 5: Rolling Statistics

def compute_rolling_features(series: pd.Series, windows=(5, 21)) -> pd.DataFrame:
    frames = {}
    for w in windows:
        # CRITICAL: .shift(1) BEFORE .rolling()
        # Without shift: rolling at t includes r_t itself → lookahead bias
        # With shift:    rolling at t covers r_{t-1} through r_{t-w} → safe
        frames[f"rolling_mean_{w}"] = series.shift(1).rolling(w).mean()
        frames[f"rolling_std_{w}"]  = series.shift(1).rolling(w).std()
    return pd.DataFrame(frames)

3.7 The Master Builder: build_feature_matrix()

def build_feature_matrix(df: pd.DataFrame) -> pd.DataFrame:
    ###############################################################
    # THE KEYSTONE LINE — shifts all price-based features by 1 day
    ###############################################################
    close = df["Close"].shift(1)

    rsi             = compute_rsi(close).rename("rsi_14")
    _, macd_signal  = compute_macd(close)
    macd_signal     = macd_signal.rename("macd_signal")
    bb_width        = compute_bollinger(close).rename("bb_width")
    lag_feats       = compute_lag_features(df["log_return"], lags=(1,2,3,5,10))
    roll_feats      = compute_rolling_features(df["log_return"], windows=(5,21))
    vix_level       = df["vix_close"].shift(1).rename("vix_level")
    vix_change      = df["vix_change_5d"].shift(1).rename("vix_change_5d")

    feature_df = pd.concat(
        [rsi, macd_signal, bb_width, lag_feats, roll_feats,
         vix_level, vix_change, df["log_return"]],   # target is LAST
        axis=1,
    )
    feature_df.dropna(inplace=True)
    return feature_df

4. The Lookahead Bias Problem

⚠ The #1 Failure Mode in Financial ML Lookahead bias occurs when your training features accidentally include information that wouldn't have been available at prediction time. The model looks brilliant in backtesting, then collapses immediately in live trading.

4.1 A Concrete Wrong Example

# ❌ WRONG — DO NOT DO THIS
df["rsi"]    = compute_rsi(df["Close"])       # uses Close_t
df["target"] = df["log_return"]              # = ln(Close_t / Close_{t-1}) — also uses Close_t!

# Both feature and target depend on Wednesday's Close.
# Model learns: "RSI is high when prices went up, prices going up = positive return"
# This is circular description, not prediction.
# Backtest result: near-perfect. Live trading result: immediate collapse.

4.2 The Correct Timeline

Features (all from t−1 or earlier)            │   Target (realized at t)
RSI( Close_{t-1}, Close_{t-2}, ... )        │   log_return_t
MACD_signal( Close_{t-1}, ... )            │   = ln( Close_t / Close_{t-1} )
lag_1 = log_return_{t-1}                   │
VIX_close_{t-1}                             │
rolling_mean_5 = mean(r_{t-1}...r_{t-5})   │
                                             │   ← Information only flows left to right

4.3 The Subtle Rolling Bug

# ❌ WRONG — rolling at t includes r_t itself
series.rolling(5).mean()

# ✅ CORRECT — window ends at t-1
series.shift(1).rolling(5).mean()

How to detect this bug: Your backtest performance degrades dramatically compared to paper trading. That gap is the fingerprint of lookahead bias.

6. Full Interview Q&A

Q1. Why do you use log returns as your prediction target instead of raw prices?
Log returns satisfy three key properties that raw prices lack. First, they are additive over time — the k-period log return is the sum of k one-period log returns, making aggregation trivial. Second, they are approximately stationary in distribution — the statistical properties of daily log returns in 2010 are comparable to 2024, unlike prices which are on completely different scales. Third, log returns arise from geometric Brownian motion, the foundation of Black-Scholes, connecting our ML pipeline to mainstream quant finance theory.
Q2. What is lookahead bias and how did you prevent it?
Lookahead bias is the temporal version of data leakage — accidentally including future information in your training features. The most common form is using the current day's price to compute an indicator, then predicting the current day's return which also depends on that price. The fix is architectural: in build_feature_matrix(), the very first step is close = df["Close"].shift(1). All technical indicators (RSI, MACD, Bollinger) are computed from this shifted series — meaning every feature at row t uses data from t-1 and earlier, never from t itself.

The subtlest case is rolling statistics. series.rolling(5).mean() at row t includes r_t — that is lookahead. The fix is series.shift(1).rolling(5).mean(), ensuring the window ends at t-1.

Q3. Why does RSI use an exponentially weighted average instead of a simple moving average?
Two reasons. First, EWM is more responsive — recent observations receive higher weight, so regime changes register faster in the RSI signal. Second, EWM avoids the "drop-off cliff" artifact. With SMA, when an extreme observation drops out of the 14-day window, RSI jumps discontinuously even if nothing happened in the market today. EWM never fully forgets an observation — it just down-weights it exponentially — so there are no artificial discontinuities in the signal.
Q4. VIX is derived from S&P 500 options, and SPY tracks the S&P 500. Aren't they the same information?
They carry different information. Rolling price volatility (like rolling_std_21) measures what has already happened — it is backward-looking realized volatility. VIX measures the options market's expectation of future volatility — it is forward-looking implied volatility. High VIX while realized price volatility is still low signals that the options market has detected an upcoming risk that the price series has not yet moved on. This divergence is an early warning signal that backward-looking features simply cannot capture.
Q5. Why did you choose SPY over individual stocks or other indices?
Three reasons. First, SPY is the most liquid ETF in the world — ~70 million shares per day — so there is no thin-trading noise that creates artificial price jumps in illiquid assets. Second, prices are efficient and well-studied, making it an honest benchmark; if your model beats buy-and-hold on SPY, it is genuinely impressive. Third, 14 years of clean daily data gives ~3,500 training examples — manageable for ML without needing high-frequency data.
Q6. What is the difference between com=, span=, and halflife= in pandas EWM, and which did you use for RSI vs. MACD?

All three are parameterizations of the same alpha (decay rate) in EWM:

  • com=k: α = 1/(1+k). Used for RSI: com=period-1=13 → α=1/14. This matches Wilder's original definition.
  • span=s: α = 2/(1+s). Used for MACD: span=12 → α≈0.154, span=26 → α≈0.074. This matches the finance industry convention for EMA-based indicators.
  • halflife=h: α = 1−e^(−ln2/h). Used when you want to specify "an observation from h days ago has half the weight of today's."
For RSI I used com=period-1 because Wilder's original 1978 formula defined the smoothing constant as 1/period, which corresponds to com=period-1 in pandas notation. For MACD I used span= because the standard 12/26/9 MACD specification was defined in terms of EMA spans.
Q7. Why 14 periods for RSI? Why not 7 or 28?
14 is the Wilder convention from 1978, representing approximately three calendar weeks of trading. It is a trade-off: shorter periods (7) are more sensitive to short-term noise and produce more false signals; longer periods (28) are too slow to react to meaningful regime changes. In practice, 14 is the industry default because it balances signal quality against responsiveness. In a research setting we would cross-validate this as a hyperparameter.
Q8. What does Bollinger Band width tell you that rolling_std_21 doesn't?
They measure the same underlying thing (realized volatility) but with two differences. First, Bollinger uses a 20-day window vs. rolling_std_21's 21-day window — minor. Second and more importantly, Bollinger Band width is normalized by the middle band (mid price), making it a relative measure: 4×std/SMA. This normalization means the value is comparable across different price levels (2010 SPY at $115 vs. 2024 at $480). rolling_std_21 is an absolute std of log returns, which is already scale-free. In practice they are correlated but not identical — Bollinger uses SMA of prices while rolling_std is computed on log returns directly.
Q9. Why use MACD signal line as your feature instead of the raw MACD line?
The raw MACD line (fast EMA minus slow EMA) is noisy. The signal line applies a 9-period EWM to the MACD line, smoothing it. For ML models, noisy features add variance to predictions without contributing predictive information — they increase the model's effective noise floor. Classic technical analysis triggers on MACD-signal crossovers, but for a continuous regression target, the smooth level of the signal line is more informative than the noisy level of the raw MACD line.
Q10. How would you detect lookahead bias if it had accidentally been introduced?
The classic detection method is comparing backtest performance to paper trading (forward-testing) performance. Lookahead bias inflates backtest metrics — directional accuracy, Sharpe ratio — dramatically. When you deploy the same model on out-of-sample data where the future is genuinely unknown, performance degrades sharply. A second method is the "correlation check": if any feature has suspiciously high correlation (|r| > 0.7) with the same-day return, you likely have a lookahead bug. Third, unit testing each feature function with a fixed seed and checking that the value at row t changes if and only if you modify data at t-1 or earlier — never if you modify data at t itself.
